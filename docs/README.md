## Who These Guides Are For

These guides were written for **computer science students** at the associate degree level.

No prior experience with web apps is assumed.

All guides use **simple language, active voice, and concrete examples**.

Jargon is explained. Complex ideas are broken into small pieces.

## Documentation Principles

This documentation follows these principles:

- ✅ **Simple Sentences** — 15–20 words per sentence
- ✅ **Active Voice** — "You create a branch" not "branches are created by developers"
- ✅ **Concrete Examples** — Every concept has code or real-world examples
- ✅ **No Jargon** — Technical terms explained when first used
- ✅ **Logical Structure** — Foundational concepts before advanced ones
- ✅ **One Idea Per Paragraph** — Easy to read and understand
